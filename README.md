# 保活Webアプリ v1.0

通常利用はGitHub Pagesなどの静的ホスティングで公開したURLを開いてください。PCでもスマホでも同じURLで利用できます。

`index.html` はGitHub Pages公開用の本体です。`app.js`、`data.js`、`style.css` と同じフォルダに置いてください。

`hoiku-web-app-v1-standalone.html` はファイル配布用の単体版です。インターネット公開せずPC内だけで使う場合に利用します。

## 使い方

1. GitHub Pagesなどで公開したURLをブラウザで開きます。
2. 既存データがある場合は、ヘッダーの「JSON読込」から `保活データ.json` を選択します。
3. 見学後、「見学評価」から各カテゴリのメモと S/A/B/C/D 評価を入力します。
4. 作業後は必ず「JSON保存」で正本ファイルを書き出します。

## GitHub Pagesで公開する場合

1. GitHubで新しいリポジトリを作成します。
2. このフォルダの中身をリポジトリ直下へアップロードします。
3. GitHubの `Settings` → `Pages` を開きます。
4. `Deploy from a branch` を選び、`main` ブランチの `/root` を指定します。
5. 表示された `https://ユーザー名.github.io/リポジトリ名/` をスマホ・PCで開きます。

## 地図表示

インターネット接続がある場合は Leaflet + OpenStreetMap で正確な地図を表示します。地図ライブラリを読み込めない環境では、緯度経度を使った簡易プロットに自動で切り替わります。

## データ方針

- 正本: 保存したJSONファイル
- localStorage: 入力途中の復元用のみ
- 保存ファイル名: `保活データ_YYYYMMDD_HHMM.json`

## 評価ロジック

- S=5, A=4, B=3, C=2, D=1
- 入園可能性 30%
- 通園運用 25%
- 保育品質 20%
- 雰囲気 15%
- 保護者負荷 5%
- 病気対応 5%
